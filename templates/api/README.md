# __APP_NAME__

A Tesl web service with a **PostgreSQL-backed entity**, cookie auth, request-body
validation, an output proof, and unit tests. Generated by `tesl init` from the
`api` template.

## Files

| file        | what it is |
|-------------|------------|
| `app.tesl`  | the application: a `Todo` entity, two routes, two proofs, two `test` blocks |
| `tesl.toml` | the project manifest (`[database].mode = "managed"`, `TESL_POSTGRES_*` env) |

## The two proofs (the point of Tesl)

1. **Input proof — `TitleSafe title`.** The `NewTodo` codec runs `isSafeTitle`
   *while decoding JSON*. By the time `createTodo` sees `newTodo.title` it is
   proven within the allowed length; a bad title is rejected with `400` before
   any handler code runs.
2. **Output proof — `Todo ? FromDb (Id == todoId)`.** `getTodo` returns a row the
   SQL `select` boundary proved came from the database under the requested id.
   The `?` return spec threads that proof to the caller, so the response is
   type-level evidence of provenance — not a value a handler could fabricate.

## Run it

```sh
tesl db up        # provision the managed local PostgreSQL ([database].mode = "managed")
tesl build        # type-check + compile (proofs and the schema are enforced here)
tesl run          # serve on $PORT (default 8086)

# in another shell:
curl -b 'user=alice' -X POST localhost:8086/todos \
  -H 'content-type: application/json' -d '{"title":"Read the Tesl tutorial"}'
curl -b 'user=alice' localhost:8086/todos/todo-1
```

## Tests

```sh
tesl test         # runs the `test "..."` blocks in app.tesl

# Run a single block by name; add --test-kind to disambiguate same-named
# blocks of different kinds (test | api-test | load-test | doctest):
tesl test --test-name "creates a todo" app.tesl
tesl test --test-name "POST /todos" --test-kind api-test app.tesl
```

To point at an existing database instead of the managed one, set
`[database].mode = "existing"` in `tesl.toml` and export the `TESL_POSTGRES_*`
variables yourself.
