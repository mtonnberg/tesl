# Tesl Todo — Elm Frontend

An Elm frontend that consumes the `todo-api.tesl` API using a generated
typed client. The Elm client generator is still experimental, so generated
API shapes and proof helpers can change between alpha releases.

## What this demonstrates

Tesl's type safety extends to the Elm frontend. The server declares facts
such as `TitleSafe` (a String between 3 and 120 characters) and `TodoId`
(a String starting with `"todo-"` with length > 5). When `tesl generate elm`
emits the client, it translates those facts into Elm proof-carrying types with
smart constructors from `mtonnberg/refinement-proofs`:

```elm
-- TitleSafe — mirrors the `isSafeTitle` check function
titleSafe : String -> Maybe (Proven String TitleSafe)

-- TodoId — mirrors the `isTodoId` check function
todoId : String -> Maybe (Proven String TodoId)
```

Because the generated smart constructors return proof-carrying values, there is
no way to pass a raw `String` to `createTodo` or `completeTodo`. The type
system enforces:

- `createTodo` takes a `NewTodo` whose `title` field is `Proven String TitleSafe` — you
  must call `titleSafe rawString` and handle the `Maybe` before constructing
  the request.
- `completeTodo` and `getTodo` take a `TodoId` — you must call
  `todoId rawString` and handle the `Maybe` before calling them.

Client-side validation runs the exact same logic as the server-side check
functions. If `titleSafe` returns `Nothing`, the string would have caused a
`400` from the server; the error is shown immediately without a network round
trip.

## Live reload workflow

`dev.sh` runs `elm reactor` and watches `todo-api.tesl` for changes together:

```
terminal 1                         terminal 2
──────────────────────────         ──────────────────────────
tesl watch example/todo-api.tesl   cd example/frontend-elm
  → recompiles & restarts server   ./dev.sh
  on every .tesl change              → regenerates TodoApi.elm on startup
                                     → on each .tesl save, regenerates
                                       again; refresh browser to pick up
```

Edit `todo-api.tesl`, save, and both the backend and `src/Api/TodoApi.elm`
update within a second. Elm's type checker catches mismatches between
`Main.elm` and the new `TodoApi.elm` on the next browser refresh.

## Running the frontend

1. Start the Tesl backend with watch (from the repo root):

   ```bash
   tesl watch example/todo-api.tesl
   ```

2. Start the dev watcher:

   ```bash
   cd example/frontend-elm
   ./dev.sh
   # elm reactor serves at http://localhost:8000
   # Open http://localhost:8000/src/Main.elm
   ```

   The API calls go to `/todos/...` — you'll need the backend running at the
   same origin, or a reverse proxy in front. For production builds:

   ```bash
   elm make src/Main.elm --output=main.js
   ```

## Manual regeneration

```bash
tesl generate elm example/todo-api.tesl --out example/frontend-elm/src/Api/TodoApi.elm
```
