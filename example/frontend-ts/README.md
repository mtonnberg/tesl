# Tesl Todo — TypeScript/Zod Frontend

A React + TypeScript frontend that consumes the `todo-api.tesl` API using a
generated Zod client. The TypeScript client generator is still experimental,
so generated names and edge-case behavior can change between alpha releases.

## What this demonstrates

Tesl's type safety extends to the frontend. The server declares facts such as
`TitleSafe` (a String between 3 and 120 characters) and `TodoId` (a String that
starts with `"todo-"` and has length > 5). When `tesl generate ts` emits the
client, it translates those facts into Zod branded types:

```typescript
// TitleSafe — mirrors the `isSafeTitle` check function
const TitleSafeSchema = z.string().min(3).max(120).brand<"TitleSafe">();

// TodoId — mirrors the `isTodoId` check function
const TodoIdSchema = z.string().refine(...).brand<"TodoId">();
```

The TypeScript compiler then enforces that:

- `createTodo` only accepts `{ title: TitleSafe }` — you cannot pass a raw
  `string` without first calling `TitleSafeSchema.parse(...)`.
- `completeTodo` and `getTodo` only accept `TodoId` — you cannot pass a plain
  `todo.id` string directly; you must parse it first.

This means format errors are caught at compile time or via client-side
validation before an API call is ever made, rather than waiting for a `400`
response from the server. Facts that cannot yet be expressed in the current
Zod emitter remain server-validated and are still marked as branded types in
the generated client.

## Live reload workflow

`vite.config.ts` includes a small plugin that watches `todo-api.tesl` and
reruns `tesl generate ts` automatically — no manual step needed:

```
terminal 1                         terminal 2
──────────────────────────         ──────────────────────────
tesl watch example/todo-api.tesl   cd example/frontend-ts
  → recompiles & restarts server   npm install && npm run dev
  on every .tesl change              → Vite starts + regenerates
                                       todo-api-client.ts on startup
                                     → on each .tesl save, client
                                       regenerates & browser HMR fires
```

Edit `todo-api.tesl`, save, and both the backend and the TypeScript client
update within a second — TypeScript errors appear immediately if the API
change breaks the frontend.

## Running the frontend

1. Start the Tesl backend with watch (from the repo root):

   ```bash
   TESL_POSTGRES_DATABASE=todo-api tesl watch example/todo-api.tesl
   # Serves on http://localhost:8086; restarts on .tesl changes
   ```

2. Install dependencies and start the dev server:

   ```bash
   cd example/frontend-ts
   npm install
   npm run dev
   # Open http://localhost:5173
   ```

   Vite proxies `/todos` to `http://localhost:8086` so no CORS configuration
   is needed during development.

## Manual regeneration

If you need to regenerate without starting the dev server:

```bash
tesl generate ts example/todo-api.tesl --out example/frontend-ts/src/todo-api-client.ts
```
