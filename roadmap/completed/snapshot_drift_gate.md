# Committed `.rkt` snapshot drift — coverage gap closed (DONE)

Completed 2026-06-30 on `core_polish`. Discovered while byte-verifying the
surface-form-lowering change (see `taken_decision.md` B5).

## The bug
Committed `.rkt` snapshots silently drifted from current compiler output:
- `example/learn/lesson32-api-tests.rkt` was **missing its entire `(module+ test …)`
  api-test block** — so the gate's Tesl-test sweep SKIPPED it (`has_test_submodule`
  greps for `(module+ test`; absent ⇒ not run). The api-tests weren't being executed.
- `example/chat/chat-backend.rkt` carried a removed `import Tesl.Env` (stale after the
  W050 cleanup) + shifted source maps; `example/kanel/KanelNotify.rkt` shifted maps.
- `lesson03-records.rkt`, `lesson41-load-tests.rkt`, `lesson32` had machine-absolute
  `/home/<user>/…` paths baked into `thsl-src!` checkpoints.

Root cause: `compile-examples.sh` (the authoritative gate) **runs** committed `.rkt`
but never regenerates+compares them, and the only regen-compare gates —
`test_integration.ml`'s exact-match and `ci.sh`'s bash loop — covered **only
`example/learn/`**. Non-learn examples (`example/*`, `example/chat`, `example/kanel`)
and `tests/*` could rot undetected.

## The fix
1. **Regenerated** every drifted committed `.rkt` (lesson32, chat-backend, KanelNotify,
   lesson03, lesson41) with repo-relative `thsl-src!` paths. A whole-corpus regen sweep
   is now clean: 126/126 example+test `.rkt` byte-identical to compiler output (only the
   3 lifted-stdlib `.rkt` differ, and those have their own `ci.sh Lifted-stdlib-snapshots`
   gate).
2. **Extended the standing exact-match gate** beyond `example/learn/`:
   `test_integration.ml` gained `tesl_files_under` (recursive walk, skips transient
   `tesl-lsp-*`), `exact_match_dir`, and two cases — `test_all_examples_exact_match`
   (all of `example/`, incl. chat/kanel) and `test_all_tests_exact_match` (`tests/`) —
   reusing the existing `assert_matches_python` (basename-normalized paths). **Proven to
   bite**: injecting one line into `example/sandbox.rkt` turns the gate RED
   (`exact output mismatch`), revert restores green.
3. **CI wiring**: `ci.sh` runs `dune test`, so the extended gate runs in CI. Deliberately
   did NOT add a regen-compare to `compile-examples.sh` — it is the authoritative gate and
   editing it to add a check that already lives (more safely) in `test_integration` risks
   the stability this whole effort is protecting. The drift gate's home is
   `test_integration` (+ `ci.sh`).

## Side effect (good)
lesson32's api-test module is now committed, so the gate's Tesl-test sweep **executes**
its api-tests (previously skipped). Net test coverage increased.

## Out of scope
Lifted-stdlib `.rkt` (`tesl/*-derived.rkt`, `list-prim.rkt`, `either.rkt`): generated by
`scripts/gen-stdlib-rkt.sh`, already drift-gated by `ci.sh`. Left to that mechanism.
