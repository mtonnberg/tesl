# S8 / C4 — independent, non-tautological emitter oracle

> Relocated 2026-07-02 from `close_all_open_issues.md` (Wave 3, item C4/S8).
> Backlog ID: **S8** (`stability_deferred_backlog.md`). Review §7, §10 item 5.

## The problem

The emitter/runtime oracle is **tautological**. Committed `.rkt` snapshots are
regenerated by `main.exe file.tesl > file.rkt` and then byte-matched against that same
compiler's output (`test_integration.ml`). A consistently-wrong-but-self-consistent
emitter passes forever — the snapshot oracle can only catch *drift* from a previously
committed emission, never a systematic emission bug. The independent reference compiler
that older comments mention no longer exists (there is no `*.py` in the tree).

Compounding this: the runtime **cannot** catch proof/capability lowering bugs by design,
because proofs are erased. A hand-applied proof-lowering corruption (`establish` a wrong
predicate) runs green.

## Why it matters

This is the single biggest assurance gap: the compiler is the sole contract, and its
output is validated only against itself. A non-tautological oracle is what converts
"trust the maintainer's emitter" into "the machine checks the emitter against an
independent semantics."

## Fix approach

Introduce a **producer-independent** oracle. Two complementary forms:

1. **Behavioral property tests** — assert observable HTTP status/body for
   capability-denial and proof-boundary paths (e.g. a capability-denied request yields
   403/500 with the expected body). Because these observe *runtime behavior* rather than
   emitted text, erasure cannot hide a lowering bug.
2. **A small independent IR interpreter** — build a Tesl IR interpreter from a committed
   behaviour table plus a grammar-driven generator, and assert `interpreter ≡
   emitted-Racket observable results`.

## Effort

**L** — a new multi-component subsystem. No expression-level IR exists today (`ir.ml` is
endpoint-focused), so the interpreter is a from-scratch build. A half-finished
interpreter is a *false oracle* worse than the byte snapshots, so this should land after
the gate-superset work (S1b / S2b-superset, now closed as C2+C3) is solid.
